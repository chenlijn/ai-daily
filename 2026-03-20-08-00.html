A Self-Evolving Framework Through Executable Subagent Accumulation and Reuse
arXiv:2603.18000 • Wed, 18 Mar 2026

Building LLM-based agents has become increasingly important. Recent works on LLM-based agent self-evolution primarily record successful experiences as textual prompts or reflections, which cannot reliably guarantee efficient task re-execution in complex scenarios.

Problem: Traditional agent self-evolution stores experiences as text, which cannot reliably re-execute tasks efficiently in complex scenarios.
Method: AgentFactory proposes preserving successful task solutions as executable subagent code that continuously refines based on execution feedback.
Results: Subagents become increasingly robust and efficient over time, progressively reducing effort for similar tasks without manual intervention.
Conclusion: Executable subagents enable continuous capability accumulation with pure Python code that ensures portability across any Python-capable system.
Enhancing LLM-Based Embodied Planning through Rule-Augmented Memory Synergy
arXiv:2603.17831 • Wed, 18 Mar 2026

LLM agents often fail in closed-world embodied environments because actions must satisfy strict preconditions and failure feedback is sparse.

Problem: LLM agents fail in embodied environments due to strict action preconditions and sparse feedback, causing invalid action generation and state drift.
Method: RPMS architecture enforces action feasibility via structured rule retrieval, gates memory applicability via belief state, and resolves conflicts via rules-first arbitration.
Results: Achieves 59.7% single-trial success with Llama 3.1 8B (+23.9 pp) and 98.5% with Claude Sonnet 4.5 (+11.9 pp); rule retrieval contributes +14.9 pp gain.
Conclusion: Episodic memory is conditionally useful—filtered by current state and constrained by explicit action rules, it becomes a stable net positive.
Governed Memory: A Production Architecture for Multi-Agent Workflows
arXiv:2603.17787 • Wed, 18 Mar 2026

Enterprise AI deploys dozens of autonomous agent nodes across workflows, each acting on the same entities with no shared memory and no common governance.

Problem: Enterprise AI faces memory silos, governance fragmentation, unstructured memories, redundant context delivery, and silent quality degradation.
Method: Governed Memory provides dual memory model, tiered governance routing, reflection-bounded retrieval, and closed-loop schema lifecycle.
Results: 99.6% fact recall, 92% governance routing precision, 50% token reduction, zero cross-entity leakage, 74.8% LoCoMo accuracy.
Conclusion: Governance and schema enforcement impose no retrieval quality penalty, enabling production multi-agent deployments at scale.
Facts as First Class Objects: Knowledge Objects for Persistent LLM Memory
arXiv:2603.17781 • Wed, 18 Mar 2026

Large language models increasingly serve as persistent knowledge workers, with in-context memory as the default strategy.

Problem: In-context memory faces capacity limits (overflow at 8,000 facts), compaction loss (60% facts destroyed), and goal drift (54% constraints eroded).
Method: Knowledge Objects (KOs) are discrete hash-addressed tuples with O(1) retrieval, serving as first-class persistent memory objects.
Results: KOs achieve 100% accuracy across all conditions at 252x lower cost; 78.9% on multi-hop reasoning versus 31.6% for in-context.
Conclusion: Density-adaptive retrieval provides a switching mechanism between in-context and KO-based memory, outperforming embedding and neural alternatives.

Problem: Standard state-value function approaches process every successor state, making them expensive to evaluate during planning.
