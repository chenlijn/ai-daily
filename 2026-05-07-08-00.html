Problem: RAG systems usually hard-code one retrieval recipe even though factoid QA, multi-hop reasoning, and scientific verification need different evidence search behavior. Method: The paper inserts an Experience-RAG skill layer between the agent and the retriever pool. It builds a scene representation from the query, dialogue history, and task metadata, looks up prior retrieval outcomes in an experience memory, routes the request to an appropriate strategy such as dense or hybrid RRF, and returns a standardized evidence package back to the agent.

Results: On a heterogeneous BeIR setup spanning NQ, HotpotQA, and SciFact, the method reaches Recall@10 0.9428, MRR@10 0.9006, and nDCG@10 0.8924. Conclusion: The useful takeaway is that retrieval strategy selection can be packaged as a reusable agent capability instead of buried inside each application workflow. That is attractive for enterprise agent stacks, although the current study is still limited to a fixed candidate pool and a fairly simple rule-based router.

Problem: Long-running agents lose coherence because flat memory files do not age, organize, or get credit-assigned properly across multi-day sessions. The paper identifies context collapse, compaction discontinuity, structural blindness, and the lack of a feedback loop from tool outcomes back into memory quality as the main reasons retrieval degrades over time.

Method: MEMTIER introduces a tiered memory stack for OpenClaw-style agents: an episodic JSONL store with provenance and cognitive weights, a five-signal retrieval engine, an attribution loop that updates memory weights from downstream use, a consolidation daemon that promotes facts into a semantic tier, and a PPO-based policy intended to tune retrieval weighting over time.

Results: On the 500-question LongMemEval-S benchmark with Qwen2.5-7B, MEMTIER reaches Acc 0.382 and F1 0.412, versus 0.050 and 0.054 for the full-context baseline. With semantic pre-population, single-session recall climbs to roughly 0.686 to 0.732, above the paper's cited RAG BM25 GPT-4o baseline of 0.560 on those categories; ablations also show semantic pre-population is the dominant gain source.

Conclusion: The practical message is that memory architecture matters more than simply throwing a larger generator at the problem. For lightweight long-running agents, structured episodic and semantic separation looks immediately actionable, while the authors also admit that dense retrieval and better attribution are still needed to solve harder multi-session synthesis.

Method: SOAR turns order allocation and robot scheduling into one event-driven RL problem. Problem: VLM agents in partially observable environments can reason about visited states, but that does not guarantee they explore the places where their world model is wrong. Method: GLANCE introduces a curiosity signal based on the mismatch between what the agent linguistically predicts and what it visually observes. A stable target network grounds visual representations, the agent uses that cross-modal discrepancy as intrinsic reward inside RL, and an adaptive curriculum reset refreshes the alignment projector so curiosity does not collapse too early in training.

Results: The paper reports that GLANCE consistently outperforms recent exploitation-heavy RL baselines such as VAGEN across several agentic tasks, including settings without dense reasoning rewards from an LLM judge. Conclusion: The deeper insight is that VLM agents need to test their own beliefs against visual reality, not just narrate them. That makes the paper relevant for embodied agents, game agents, and future multimodal assistants that must explore uncertain environments rather than merely react to them.

Standard parametric memory forgets and lacks interpretability, while retrieval-based memory struggles to align heterogeneous modalities efficiently on edge hardware.

Method: ScrapMem organizes heterogeneous memories into scrapbook-style multimodal pages, then applies optical forgetting that progressively lowers the resolution of older memories to preserve semantics while cutting storage. An episodic memory graph adds causal and temporal structure so the system can retrieve and reason over user history more coherently.

Results: On ATM-Bench, ScrapMem reaches a new state of the art with Joint@10 of 51.0 percent, ahead of ATM-RAG at 48.6 percent, and improves Recall@10 to 70.3 percent. Conclusion: This is a practical memory paper for privacy-sensitive edge agents. The combination of multimodal aggregation plus controlled degradation is appealing because it trades off fidelity and storage explicitly instead of pretending every old memory deserves full-resolution retention forever.

10. QKVShare: Quantized KV-Cache Handoff for Multi-Agent On-Device LLMs

arXiv: 2605.03884 2026-05-07 Heat Score: 65 (Watchlist) HF Trending: No | HF Upvotes: 0

Why it matters now: It matches today’s hot direction(s): agent, efficiency, generation. Its topic resonates with today’s trending model activity on Hugging Face. Its direction is also showing up in today’s GitHub tooling and implementation activity. It overlaps with what the builder community is actively discussing today.

Problem: When one on-device agent hands work to another, the receiver usually rebuilds context from scratch or receives an expensive full-precision KV cache. Both options are painful on edge hardware, where KV memory grows with context length and can dominate latency and memory budgets.

Method: QKVShare packages a sender's KV state into a quantized CacheCard, assigns bits with a token-level mixed-precision controller, and injects the received cache directly into the next model through a HuggingFace-compatible path. The design explicitly studies both accuracy under repeated multi-hop handoff and the latency breakdown after cache injection.

Results: On 150 GSM8K problems with Llama-3.1-8B-Instruct, adaptive quantization stays competitive under repeated handoff and helps most in deeper-hop, higher-budget settings. For latency, the handoff path reduces time-to-first-token versus full re-prefill at every tested context, from 130.7 ms versus 150.2 ms at nominal 1K context to 397.1 ms versus 1029.7 ms at 8K context.

Conclusion: The paper gives credible evidence that quantized latent handoff is a real systems primitive for multi-agent edge inference. It is still early, especially around controller ablations, but the measured TTFT gains make this a direction worth watching for mobile or low-memory multi-agent deployments.
