📄 Top 10 arXiv Papers

1. How Much LLM Does a Self-Revising Agent Actually Need?

arXiv: 2604.07236 2026-04-09

Problem: Understanding what portion of LLM-based agent competence comes from the language model itself versus explicit structure around it has been difficult to measure.

Method: The authors introduce a reflective runtime protocol that externalizes agent state, confidence signals, guarded actions, and hypothetical transitions into inspectable structure. They evaluate on Collaborative Battleship using four progressively structured agents over 54 games.

Results: Explicit world-model planning improves +24.1pp win rate over greedy baseline. Adding conditional LLM revision at 4.3% of turns yields only marginal F1 improvement (+0.005) with win rate dropping (31 to 29 out of 54).

Conclusion: Externalizing reflection turns latent agent behavior into inspectable runtime structure, allowing direct study of LLM intervention's marginal role. The explicit structure contributes more than previously assumed.

2. Reason in Chains, Learn in Trees: Self-Rectification and Grafting for Multi-turn Agent Policy Optimization

arXiv: 2604.07165 2026-04-08

Problem: Reinforcement learning for LLM agents suffers from sparse rewards in multi-step reasoning tasks, where critical steps are drowned out by less important ones.

Method: T-STAR (Tree-structured Self-Taught Agent Rectification) consolidates trajectories into a unified Cognitive Tree by merging functionally similar nodes. It uses Introspective Valuation for variance-reduced advantage estimation and In-Context Thought Grafting to synthesize corrective reasoning.

Results: T-STAR achieves consistent improvements over strong baselines across embodied, interactive, reasoning, and planning benchmarks, with largest gains on tasks requiring extended reasoning chains.

Conclusion: The Cognitive Tree structure effectively addresses sparse reward problems by identifying and amplifying critical decision points in multi-step reasoning.

3. Benchmarking LLMs on Dynamic, Multi-Objective Geo-Spatial Exploration

arXiv: 2604.07070 2026-04-08

Problem: Existing Geo-Spatial QA benchmarks focus on static retrieval, failing to capture real-world planning with dynamic user locations and compound constraints.

Method: EVGeoQA benchmark built on EV charging scenarios with location-anchored dual-objective design. Proposes GeoRover framework using tool-augmented agent architecture to evaluate LLMs.

Results: LLMs successfully utilize tools for sub-tasks but struggle with long-range spatial exploration. An emergent capability was observed: LLMs can summarize historical trajectories to enhance exploration efficiency.

Conclusion: EVGeoQA establishes a challenging testbed for future geo-spatial intelligence research, revealing gaps in current LLM planning abilities.
