Problem: Most agent benchmarks test short terminal tasks and grade only the final answer, so they miss partial progress, debugging ability, and long-horizon planning failures. This paper targets the harder regime where agents must work for minutes to hours across open-ended software, scientific, and multimodal tasks with meaningful intermediate steps.

Method: The authors build Long-Horizon-Terminal-Bench, a 46-task benchmark across nine categories. Each task keeps the terminal-style executable setup of prior work, but is decomposed into fine-grained subtasks with dense reward-based grading so evaluators can measure how far an agent gets, not just whether it fully solves the task.

Results: Across 15 frontier models, runs average 9.9M tokens, 231 episodes, and 85.3 minutes per task, which makes the benchmark substantially harder than earlier terminal suites. Even the best model reaches only 15.2% pass@1 at a 0.95 partial-reward threshold and 10.9% at a perfect 1.0 threshold; the mean across models is 4.3% and 1.7% respectively.

Conclusion: This is a useful benchmark paper for anyone building coding or tool-using agents because it exposes how weak current systems still are once tasks become long, stateful, and iterative. The dense-grading design is especially valuable for post-training and evaluation because it can support finer reward signals than pass/fail tests.

Problem: Code-generating agent systems repeatedly lose useful context between sessions, but simply storing full chat histories makes generation noisy, stale, and expensive. The paper focuses on enterprise-style workflows where agents need to preserve schemas, constraints, and tool settings without dragging along irrelevant reasoning traces.

Method: It proposes shared selective persistent memory, which stores four reusable context classes: task specifications, data schemas, tool configurations, and output constraints. The memory is packaged in shareable workspaces with role-based access control, and paired with a zero-token data refresh mechanism so generated artifacts can be updated from fresh runtime data without re-invoking the LLM.

Results: In three enterprise scenarios, the system reaches 96% task completion versus 79% without memory and 71% with naive full-history persistence. Zero-token refresh cuts recurring update time by 14x, summary-driven generation reduces token cost by 97x versus raw-data injection, and a replication on four public datasets succeeds in 12/12 zero-refresh trials.

Conclusion: The practical message is strong: selective memory beats both forgetting everything and keeping everything. For enterprise agent products, this points toward shared structured memory and executable artifacts as a better design pattern than ever-growing conversation logs.
