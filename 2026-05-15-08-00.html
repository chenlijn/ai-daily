1. MMSkills: Towards Multimodal Skills for General Visual Agents

arXiv: 2605.13527 2026-05-15 Heat Score: 87 (High) HF Trending: No | HF Upvotes: 0

Why it matters now: This is the strongest cross-signal paper of the day because it sits at the intersection of visual agents, reusable skills, and inference efficiency, all of which are echoed by today's agent-focused GitHub projects and multimodal model releases. For Charlie's interests, it is especially relevant because it turns multimodal procedural knowledge into something reusable rather than prompt-only.

Problem: Most agent skill libraries store reusable behavior as text prompts, code, or routines, which works poorly once an agent must read screens, notice UI state, and verify progress visually. The paper targets the gap between knowing the next action in words and actually grounding that action in pixels for GUI and game agents.

Method: MMSkills packages a skill as a textual procedure plus runtime state cards and multi-view keyframes. A trajectory-to-skill generator mines public interaction traces, groups workflows, induces procedures, grounds them visually, and audits the result; at inference time a branch-loaded skill agent opens a temporary skill branch, aligns the stored visual evidence with the live screen, and returns structured guidance such as applicability, subgoal, plan, failure cues, and verification checks.

Results: The paper reports consistent gains on GUI and game-based visual-agent benchmarks for both frontier and smaller multimodal agents. The visible text does not expose every table value in the current extraction, but the reported trend is that multimodal skill packages beat text-only reuse and improve decision quality without requiring full-image context at every step.

Conclusion: The key contribution is showing that reusable agent experience for visual tasks should be stored as multimodal procedural knowledge, not just text. This is useful for desktop automation, embodied UI workflows, and any enterprise agent that must recover from visually observable state changes.

2. IdeaForge: A Knowledge Graph-Grounded Multi-Agent Framework for Cross-Methodology Innovation Analysis and Patent Claim Generation

arXiv: 2605.13311 2026-05-15 Heat Score: 84 (High) HF Trending: No | HF Upvotes: 0

Why it matters now: IdeaForge ranks high because it connects multi-agent orchestration with an enterprise-facing outcome, patent claim generation, and uses a graph backbone for traceability. That combination resonates with today's broader shift from chat demos to auditable workflow systems.

Problem: Single-methodology ideation pipelines such as TRIZ-only or Design-Thinking-only prompting tend to lose intermediate reasoning structure and produce fragmented innovation suggestions. That makes it hard to trace why a claim was generated, compare evidence across methods, or score novelty systematically in high-stakes invention workflows.

Method: IdeaForge runs specialist agents for TRIZ, Design Thinking, and SCAMPER over a persistent FalkorDB knowledge graph. Each agent writes structured entities and relations, then a convergence layer links claims supported by multiple methodologies through CONVERGENT edges; an InnovationScore ranks candidates using convergence support, method diversity, claim strength, and prior-art challenges before a drafting agent turns the best subgraphs into patent-style claims.

Results: On a legal-technology use case, the authors report that graph-grounded cross-method synthesis produces more diverse and more traceable innovation candidates than single-method baselines. The accessible extraction does not expose full numeric deltas, so the safest reading is qualitative improvement in diversity, traceability, and confidence ranking rather than a published headline percentage.

Conclusion: The paper matters because it treats AI ideation as a structured evidence-convergence problem rather than free-form prompting. For enterprise R&D, IP teams, and regulated innovation workflows, the graph-native design is the interesting part because it gives an audit trail instead of just a polished final paragraph.

3. Harnessing Agentic Evolution

arXiv: 2605.13821 2026-05-15 Heat Score: 81 (High) HF Trending: No | HF Upvotes: 0

Why it matters now: AEvo is timely because agent builders are moving from one-shot prompting to iterative self-improvement loops, and this paper asks how to improve the improvement process itself. It also connects neatly to today's agent tooling wave on GitHub, where orchestration and evaluation are becoming first-class concerns.

Problem: Existing agentic evolution systems are either rigid hand-designed procedures or flexible free-form agents that drift over long horizons. Both accumulate useful evidence, including failed candidates, traces, and feedback, but they lack a stable interface for revising the mechanism that guides future search.

Method: The paper formulates evolution as an interactive environment whose state is the accumulated search context. AEvo adds a meta-agent that does not directly propose the next candidate; instead it edits the procedure, prompts, or agent context that will govern subsequent search, giving one unified interface for both procedure-based and agent-based evolution.

Results: Empirically, AEvo outperforms five evolution baselines on agentic and reasoning benchmarks, with a reported 26 relative improvement over the strongest baseline. On three open-ended optimization tasks it also beats four baselines and reaches state-of-the-art performance under the same iteration budget.

Conclusion: The practical lesson is that long-horizon agent optimization benefits from explicit meta-control over the search process, not just better candidate generation. This has clear value for coding agents, research agents, and auto-tuning workflows where evidence accumulates over many rounds.

4. RS-Claw: Progressive Active Tool Exploration via Hierarchical Skill Trees for Remote Sensing Agents

arXiv: 2605.13391 2026-05-15 Heat Score: 79 (High) HF Trending: No | HF Upvotes: 0

Why it matters now: RS-Claw is one of the clearest engineering papers of the day because it attacks tool overload in real agent systems and reports a very tangible token-compression result. It also aligns with the current industry obsession with context budgeting and tool routing for large agent stacks.

Problem: Remote-sensing agents often face very large heterogeneous toolboxes, and today's passive approaches either register everything up front or retrieve a subset with RAG. Full registration wastes context on irrelevant tools, while retrieval can miss the one critical operator needed at a key reasoning step.

Method: RS-Claw organizes tools into a hierarchical skill tree. The agent first reads only high-level summaries to choose promising branches, then progressively loads more detailed descriptions and finally invokes concrete tools, effectively turning tool selection into active exploration instead of one-shot retrieval.

Results: On the Earth-Bench benchmark, the paper reports up to 86 percent input-token compression while outperforming Flat and RAG baselines on complex reasoning tasks. The central empirical claim is that hierarchical exploration reduces semantic noise and preserves access to the right specialist tools during long-horizon reasoning.

Conclusion: The broader message is that tool-use quality depends heavily on how the tool space is exposed to the model. For enterprise copilots with hundreds of APIs or skills, progressive loading could be a more scalable pattern than dumping everything into context or trusting a single retrieval step.

5. Senses Wide Shut: A Representation-Action Gap in Omnimodal LLMs

arXiv: 2605.13737 2026-05-15 Heat Score: 73 (Medium) HF Trending: No | HF Upvotes: 0

Why it matters now: This paper stands out because omnimodal models are being marketed as grounded agents, yet the authors show a sharp gap between internal perception and final action. That is an important warning signal for any production workflow that relies on video, audio, and text jointly.

Problem: The authors test a basic but underexplored grounding failure: when the text premise contradicts what the model actually sees or hears, will the model reject the false premise. Existing multimodal benchmarks mostly measure comprehension, not whether a model can notice and act on premise-perception conflicts.

Method: They build IMAVB, a 500-clip long-form movie benchmark with a 2x2 design over target modality and premise condition. The study probes hidden states for mismatch signals, evaluates behavior across eight open-source omnimodal LLMs plus Gemini 3.1 Pro, and introduces a probe-guided logit adjustment that injects the detected conflict signal back into decoding.

Results: The main finding is a representation-action gap: hidden states often encode the mismatch even when outputs fail to reject the false claim. The effect is prompt-resistant across seven prompt variants, audio grounding lags vision, and the probe-guided adjustment consistently improves rejection behavior, though the current extraction does not surface every exact metric from the tables.

Conclusion: The paper argues that the bottleneck in omnimodal grounding may lie in translating perception into action rather than in perception itself. For safety reviews, multimodal assistants, and video-analysis systems, that means output calibration and decoding control deserve as much attention as encoder quality.

6. TRIAGE: Evaluating Prospective Metacognitive Control in LLMs under Resource Constraints

arXiv: 2605.13414 2026-05-15 Heat Score: 69 (Medium) HF Trending: No | HF Upvotes: 0

Why it matters now: TRIAGE deserves attention because compute allocation is becoming an agent capability, not just an infrastructure concern. As token budgets and reasoning modes get more expensive, models that know when to spend compute become materially more deployable.

Problem: An autonomous model facing a queue of tasks under a fixed token budget must decide what to attempt, in what order, and how much compute to allocate before seeing any execution feedback. Standard benchmarks mostly test task accuracy, so they miss this prospective metacognitive control problem.

Method: TRIAGE gives the model a pool of tasks and a token budget calibrated to its own baseline cost, then requires one ordered plan encoding selection, sequencing, and per-task allocation. Plans are scored against an oracle with full knowledge of solvability and cost, producing a triage efficiency ratio that can compare models on a common scale.

Results: The evaluation spans competition math, graduate science, code generation, and expert knowledge tasks across frontier and open-source models with and without reasoning mode. The paper finds substantial gaps between current model behavior and the oracle, indicating that today's LLMs are still weak at proactively managing scarce compute rather than merely answering individual questions.

Conclusion: This is a useful benchmark contribution because it reframes “smartness” as resource-aware planning under budget. For production agents, that directly maps to latency control, bounded-cost workflows, and deciding when expensive reasoning is actually worth it.

7. Ego2World: Compiling Egocentric Cooking Videos into Executable Worlds for Belief-State Planning

arXiv: 2605.13335 2026-05-15 Heat Score: 68 (Medium) HF Trending: No | HF Upvotes: 0

Why it matters now: Ego2World earns its place because it pushes embodied-agent evaluation closer to real human activity while keeping executable structure for planning. That mix of egocentric video and belief-state planning is a strong signal for next-wave household and robotic agents.

Problem: Household agents must act under partial observation, remember hidden object states, and recover after failed actions, but many simulators assume overly clean dynamics or full observability. Egocentric video datasets are realistic but passive, so they do not by themselves test executable planning and memory maintenance.

Method: The authors compile egocentric cooking videos from HD-EPIC into executable symbolic worlds with graph-transition rules derived from annotations. During evaluation, the simulator keeps the true hidden world graph while the agent only maintains a partial belief graph from local observations and execution feedback, forcing explicit memory updates and replanning.

Results: The paper reports that simple action-overlap metrics overestimate real physical-state success. Persistent belief memory improves task completion and reduces repeated mistakes, showing that world-state tracking matters materially once the benchmark hides the ground-truth state from the agent.

Conclusion: Ego2World is interesting because it creates a middle ground between passive video understanding and synthetic embodied simulators. For robotics and household-assistant research, it offers a more realistic test of memory, recovery, and long-horizon planning.

Problem: As GenAI tools become common in education, instructors need to know whether machine-generated slides are accurate, complete, and pedagogically sound, and whether students react differently to them. Method: The study compares an end-to-end education tool, two general LLM assistants, and two coding assistants for generating slides from instructor-authored notes. Results: Coding-assistant tools produce the strongest slides in the authors' evaluation, especially on accuracy, completeness, and pedagogy. Students rate AI-generated slides similarly to human-authored ones and cannot reliably identify which decks were AI-generated; the paper also finds a negative correlation between slide quality ratings and the belief that the slides were AI-made.

Conclusion: The takeaway is not that slide generation is solved, but that well-chosen AI workflows are already credible for internal training and course-support tasks. For Charlie's marketing and enablement use cases, it reinforces that content-assembly pipelines may benefit more from strong prompting plus editing than from specialized slide-only systems.

9. Scaling Retrieval-Augmented Reasoning with Parallel Search and Explicit Merging

arXiv: 2605.13534 2026-05-15 Heat Score: 65 (Watchlist) HF Trending: No | HF Upvotes: 0

Why it matters now: MultiSearch is timely because deep-search and RAG agents are a major product pattern right now, yet most systems still retrieve through one query at a time. The paper speaks directly to the accuracy-versus-noise tradeoff that shows up in enterprise search and long-form reasoning products.

Problem: Single-query retrieval at each reasoning step can miss relevant evidence, amplify noise, and force the model into extra reasoning turns. This becomes a bottleneck for deep-search agents that need broad coverage but also clean evidence aggregation.

Method: MultiSearch generates several queries from different perspectives at each reasoning step, retrieves information in parallel, and then explicitly merges and refines the evidence before continuing reasoning. The framework is trained with reinforcement learning and a multi-process reward that jointly optimizes query diversity, retrieval quality, and the merge stage.

Results: Across seven benchmarks, the paper reports better reasoning accuracy than prior deep-search or single-query retrieval baselines. The visible extract does not expose every benchmark number, so the conservative summary is that the gains come from higher information coverage plus cleaner evidence consolidation rather than from longer chains of thought alone.

Conclusion: The engineering lesson is that retrieval quality depends on search breadth and merge discipline together. This is directly relevant to enterprise knowledge assistants, report-generation agents, and any system where one brittle query can hide the decisive evidence.

10. Cognifold: Always-On Proactive Memory via Cognitive Folding

arXiv: 2605.13438 2026-05-15 Heat Score: 63 (Watchlist) HF Trending: No | HF Upvotes: 0

Why it matters now: Cognifold rounds out the top 10 because agent memory is becoming a core product differentiator, and this work pushes beyond on-demand retrieval toward always-on structure formation. That is highly aligned with current interest in proactive assistants and long-lived agent sessions.

Problem: Most agent-memory systems are reactive: they retrieve past items when asked but do not continuously reorganize experience into durable cognitive structure. That limits proactive behavior, intent formation, and long-horizon continuity across fragmented event streams.

Method: Cognifold extends complementary learning systems from a two-layer memory view to a three-layer design with a prefrontal intent layer. Its CogniFold mechanism continuously folds incoming events into self-organizing graph structures that merge when semantically similar, decay when stale, reconnect through associative recall, and surface intents when concept density crosses a threshold.

Results: Using CogEval-Bench, the paper reports that CogniFold forms memory structures that better match the target cognitive expectations than the compared baselines. The available extraction does not reveal all metric values, but the claimed advantage is structural quality and proactive organization rather than only retrieval hit rate.

Conclusion: The paper is valuable because it reframes memory as an always-running organizational process instead of a lookup table. For personal assistants and enterprise copilots, that could eventually support better task carryover, proactive reminders, and richer user-state modeling.
