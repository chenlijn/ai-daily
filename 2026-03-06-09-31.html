1. The Spike, the Sparse and the Sink: Anatomy of Massive Activations and Attention Sinks

https://arxiv.org/abs/2603.05498

Architecture Mechanistic Interpretability

Problem Understanding two recurring phenomena in Transformers: massive activations (extreme outliers in few channels) and attention sinks (tokens attracting disproportionate attention) - their relationship and functional roles were unclear.
Method Systematic experiments analyzing the co-occurrence patterns, identifying pre-norm configuration as the key architectural choice enabling co-occurrence. Showed the phenomena serve related but distinct functions through ablation studies.
Results Massive activations function as implicit parameters (global, near-constant hidden representations), while attention sinks modulate outputs locally (bias toward short-range dependencies). Ablating pre-norm causes decoupling of these phenomena.
Conclusion The co-occurrence is largely an architectural artifact of modern Transformer design, not a functional necessity. This insight enables more targeted model compression and optimization strategies.

2. Towards Provably Unbiased LLM Judges via Bias-Bounded Evaluation

https://arxiv.org/abs/2603.05485

Evaluation LLM-as-Judge

Problem As AI systems move toward autonomous self-maintaining feedback loops, we need automated verifiable rewards. LLM judges lack systems capable of enforcing standards with strong guarantees, especially when bias vectors are unknown or adversarial.
Method Proposed A-BB (Average Bias-Boundedness), an algorithmic framework formally guaranteeing reductions of harm from any measurable bias in LLM judges. Tested on Arena-Hard-Auto with four LLM judges.
Results Achieved (tau=0.5, delta=0.01) bias-bounded guarantees while retaining 61-99% correlation with original rankings. Most judge-bias combinations exceeded 80% correlation retention across formatting and schematic bias settings.
Conclusion Formal bias-bounded guarantees are achievable without sacrificing ranking fidelity. This framework enables safer deployment of LLM judges in high-stakes autonomous systems requiring verifiable feedback.

3. Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering

https://arxiv.org/abs/2603.05344

Code Agents CLI OPENDEV

Problem AI coding assistance is shifting from IDE plugins to terminal-native agents, but effective autonomous assistance requires strict safety controls and efficient context management to prevent bloat and reasoning degradation in long-horizon tasks.
Method OPENDEV uses compound AI system architecture with workload-specialized model routing, dual-agent architecture (planning vs execution separation), lazy tool discovery, adaptive context compaction, and automated memory system for project knowledge accumulation.
Results Enforces explicit reasoning phases and context efficiency. The system provides secure, extensible foundation for terminal-first AI assistance with event-driven system reminders to counteract instruction fade-out.
Conclusion Terminal-native agents represent a paradigm shift offering unprecedented autonomy. Success requires architectural innovations in context management, safety controls, and memory systems - providing a blueprint for robust autonomous software engineering.

4. Judge Reliability Harness: Stress Testing the Reliability of LLM Judges

https://arxiv.org/abs/2603.05399

Evaluation Reliability Testing

Problem As LLM-based scoring is widely deployed in AI benchmarks, more tooling is needed to efficiently assess the reliability of these methods. Judges show consistency issues due to simple text formatting changes, paraphrasing, verbosity changes, and ground truth label flipping.
Method Open-source library for constructing validation suites testing LLM judge reliability. Evaluates binary judgment accuracy and ordinal grading for free-response and agentic tasks. Tested four state-of-the-art judges across four benchmarks (safety, persuasion, misuse, agentic behavior).
Results Found meaningful variation across models and perturbation types. No judge evaluated was uniformly reliable across benchmarks using the harness. Consistency issues persist even with simple formatting perturbations.
Conclusion Systematic stress testing reveals that current LLM judges have significant robustness gaps. The harness provides practical tooling for researchers and practitioners to evaluate and improve judge reliability before production deployment.

5. WebChain: A Large-Scale Human-Annotated Dataset of Real-World Web Interaction Traces

https://arxiv.org/abs/2603.05295

Web Agents Dataset Multimodal

Problem Web agent research lacks large-scale, reproducible datasets of real-world web interactions. Synthetic methods often miss complex, high-value tasks critical for training robust agents.
Method WebChain contains 31,725 trajectories and 318k steps with Triple Alignment of visual, structural, and action data. Scalable pipeline ensures coverage of complex tasks. Proposed Dual Mid-Training recipe decoupling spatial grounding from planning.
Results Achieved state-of-the-art performance on WebChainBench and other public GUI benchmarks. Largest open-source dataset of its kind designed for reproducible web agent research.
Conclusion High-quality, diverse real-world interaction data is essential for next-generation web agents. The Dual Mid-Training approach of separating grounding from planning provides a recipe for building more capable agents.

6. STRUCTUREDAGENT: Planning with AND/OR Trees for Long-Horizon Web Tasks

https://arxiv.org/abs/2603.05294

Web Agents Planning Hierarchical

Problem Existing web agents struggle on complex, long-horizon tasks due to limited in-context memory for tracking history, weak planning abilities, and greedy behaviors leading to premature termination.
Method Hierarchical planning framework with: (1) online hierarchical planner using dynamic AND/OR trees for efficient search, (2) structured memory module tracking candidate solutions for constraint satisfaction in information-seeking tasks. Produces interpretable hierarchical plans.
Results Improved performance on WebVoyager, WebArena, and custom shopping benchmarks compared to standard LLM-based agents. Enables easier debugging and human intervention through interpretable plans.
Conclusion Explicit hierarchical planning with structured memory addresses the core limitations of current web agents. AND/OR tree representations provide both efficiency and interpretability for long-horizon task completion.

7. Mapping LLM Reasoning Capability via Formalized and Calibrated Probes (X-RAY)

https://arxiv.org/abs/2603.05290

Reasoning Evaluation Formal Methods

Problem LLM reasoning capability remains poorly understood. Existing evaluations emphasize task-level accuracy, often conflating pattern matching with true reasoning. Need contamination-free evaluation supporting training and testing.
Method X-RAY system maps reasoning via calibrated, formally verified probes. Models reasoning as function of extractable structure through formal properties (constraint interaction, reasoning depth, solution-space geometry). Generates probes via formal tools with controlled structural variations.
Results Revealed systematic asymmetry: models robust to constraint refinement (additional conditions shrink solution space) but degrade sharply under solution-space restructuring. Differentiated models indistinguishable on standard benchmarks. Revealed structurally interpretable failure modes.
Conclusion Formal, calibrated probes reveal fundamental reasoning limitations invisible to standard benchmarks. The framework enables precise isolation of structural information and contamination-free training/testing of reasoning models.

8. PACE: A Personalized Adaptive Curriculum Engine for 9-1-1 Call-taker Training

https://arxiv.org/abs/2603.05361

Education Personalization Adaptive Systems

Problem 9-1-1 call-taking requires mastery of 1000+ interdependent skills. Nationwide labor shortage strains training capacity, but effective instruction requires personalization to each trainee's evolving competencies - a burden current practice cannot scale.
Method PACE co-pilot system: (1) maintains probabilistic beliefs over trainee skill states, (2) models individual learning/forgetting dynamics, (3) recommends scenarios balancing new competency acquisition with retention. Uses skill graph for diagnostic coverage and contextual bandits for scenario selection.
Results 19.50% faster time-to-competence and 10.95% higher terminal mastery vs state-of-the-art. 95.45% alignment between PACE and expert pedagogical judgments. Reduced turnaround time from 11.58 minutes to 34 seconds (95.08% reduction).
Conclusion AI-driven adaptive curriculum engines can dramatically accelerate training while maintaining quality. Probabilistic skill modeling combined with contextual bandits provides an effective framework for personalized education at scale.

9. A Psyche-Driven Chess Engine with Dynamic Emotional Modulation

https://arxiv.org/abs/2603.05352

Game AI Human-like Behavior Psychology

Problem Chess engines surpassed human strength but don't play like humans. Grandmasters under pressure blunder differently than club players on streaks. Conventional engines capture none of this behavioral variability.
Method Personality x Psyche decomposition: Personality is static preset; Psyche is dynamic scalar ψ_t ∈ [-100, +100] recomputed from five positional factors after every move. Audio-inspired signal chain (noise gate, compressor, equalizer, limiter) reshapes move probabilities on the fly.
Results Tested across 12,414 games. Both probability sources showed same monotonic gradient in top-move agreement (~20-25 pp spread from stress to overconfidence). Overconfident: 66% agreement with vanilla Maia2. Under stress: competitive score fell from 50.8% to 30.1%.
Conclusion Behavioral variation comes from the signal chain, not the underlying model. The framework successfully mimics human-like tilt and overconfidence patterns. Engine-agnostic approach works with any probability-outputting system.

10. Reclaiming Lost Text Layers for Source-Free Cross-Domain Few-Shot Learning

https://arxiv.org/abs/2603.05235

Computer Vision CLIP CVPR 2026

Problem SF-CDFSL (Source-Free Cross-Domain Few-Shot Learning) focuses on fine-tuning with limited target domain data. CLIP shows promise but removing certain middle text encoder layers improves performance - these "Lost Layers" seem redundant but their information is actually beneficial.
Method Discovered visual gaps prevent useful layer information from being fully utilized. Proposed method to re-utilize lost layer information at layer and encoder levels, guiding visual branch re-learning under domain shifts. Addresses underutilized text encoder information.
Results Extensive experiments across CLIP, SigLip, PE-Core backbones and 4 CDFSL datasets + 10 Meta-dataset datasets. Method effectively addresses underutilized information issue, outperforming simple layer removal approaches.
Conclusion Text encoder layers deemed "lost" actually contain valuable information blocked by visual gaps. Teaching the model to reclaim this information enables better cross-domain few-shot learning without requiring source data access.
