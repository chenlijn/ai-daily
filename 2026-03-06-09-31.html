1. The Spike, the Sparse and the Sink: Anatomy of Massive Activations and Attention Sinks

arXiv:2603.05498 | Mar 5, 2026
https://arxiv.org/abs/2603.05498
Problem: Transformer language models exhibit two puzzling phenomena: massive activations (extreme outliers in token representations) and attention sinks (certain tokens attract disproportionate attention). Their functional roles and relationship remain unclear.
Method: Systematic experiments analyzing the co-occurrence of massive activations and attention sinks. Identified pre-norm configuration as the key architectural choice enabling this co-occurrence. Massive activations function as implicit global parameters, while attention sinks modulate local attention outputs.
Results: Shows that co-occurrence is largely an architectural artifact. Ablating pre-norm causes the two phenomena to decouple. Massive activations induce near-constant hidden representations across layers.
Conclusion: These phenomena serve related but distinct functions. Understanding their architectural origins provides insights into Transformer design and could inform more efficient model architectures.

2. Towards Provably Unbiased LLM Judges via Bias-Bounded Evaluation

arXiv:2603.05485 | Mar 5, 2026
https://arxiv.org/abs/2603.05485
Problem: As AI systems move toward autonomous feedback loops, we need LLM judges with strong guarantees against bias, particularly when bias vectors are unknown or adversarially discovered. Current LLM judges lack formal guarantees.
Method: Proposed average bias-boundedness (A-BB), an algorithmic framework that formally guarantees reductions of harm from any measurable bias in an LLM judge. Evaluated on Arena-Hard-Auto with four LLM judges.
Results: Achieved (tau=0.5, delta=0.01) bias-bounded guarantees while retaining 61-99% correlation with original rankings. Most judge-bias combinations exceeded 80% correlation retention.
Conclusion: First framework to provide formal bias-bounded guarantees for LLM judges. Critical for building trustworthy autonomous AI systems that depend on automated, verifiable rewards and feedback.

3. Building AI Coding Agents for the Terminal: Scaffolding, Harness, Context Engineering, and Lessons Learned

arXiv:2603.05344 | Mar 5, 2026
https://arxiv.org/abs/2603.05344
Problem: AI coding assistance is shifting from IDE plugins to terminal-native agents, but these agents need strict safety controls and efficient context management to prevent context bloat and reasoning degradation.
Method: Introduced OPENDEV, a CLI-based coding agent with compound AI system architecture. Features: workload-specialized model routing, dual-agent architecture (planning vs execution), lazy tool discovery, adaptive context compaction, and automated memory system.
Results: Successfully addresses long-horizon development tasks with context efficiency. Prevents instruction fade-out through event-driven system reminders. Provides secure, extensible foundation for terminal-first AI assistance.
Conclusion: Terminal-native agents represent a fundamental shift in AI coding assistance. The paper provides a blueprint for robust autonomous software engineering with explicit reasoning phases and context efficiency.

4. Judge Reliability Harness: Stress Testing the Reliability of LLM Judges

arXiv:2603.05399 | Mar 5, 2026 | ICLR 2026 Workshop
https://arxiv.org/abs/2603.05399
Problem: LLM-based scoring is widely deployed in AI benchmarks, but insufficient tooling exists to assess the reliability of these methods. Simple perturbations like formatting changes can affect judgment consistency.
Method: Open-source library for constructing validation suites testing LLM judge reliability. Generates reliability tests evaluating binary judgment accuracy and ordinal grading for free-response and agentic tasks. Tested across safety, persuasion, misuse, and agentic behavior benchmarks.
Results: No evaluated judge is uniformly reliable across benchmarks. Found meaningful variation in performance across models and perturbation types (text formatting changes, paraphrasing, verbosity changes, ground truth label flipping).
Conclusion: Highlights critical need for improved robustness in LLM judges. Provides practical tool for researchers to evaluate and improve judge reliability before deployment in autonomous systems.

https://arxiv.org/abs/2603.05218
Problem: Enterprise search requires agents capable of complex, hard-to-verify agentic search tasks across diverse information sources. Traditional approaches lack generalization across heterogeneous search behaviors.

7. STRUCTUREDAGENT: Planning with AND/OR Trees for Long-Horizon Web Tasks

arXiv:2603.05294 | Mar 5, 2026
https://arxiv.org/abs/2603.05294
Problem: Existing web agents struggle on complex, long-horizon tasks due to limited in-context memory, weak planning abilities, and greedy behaviors leading to premature termination.
Method: Hierarchical planning framework with: (1) online hierarchical planner using dynamic AND/OR trees for efficient search, and (2) structured memory module tracking candidate solutions to improve constraint satisfaction in information-seeking tasks.
Results: Improved performance on long-horizon web-browsing tasks on WebVoyager, WebArena, and custom shopping benchmarks. Produces interpretable hierarchical plans enabling easier debugging and human intervention.
Conclusion: Structured planning with explicit AND/OR trees significantly improves agent performance on complex web tasks. Demonstrates importance of hierarchical reasoning for autonomous web agents.

8. X-RAY: Mapping LLM Reasoning Capability via Formalized and Calibrated Probes

arXiv:2603.05290 | Mar 5, 2026
https://arxiv.org/abs/2603.05290
Problem: Existing evaluations conflate pattern matching with true reasoning capability. Need to understand how LLMs reason at a structural level, not just task-level accuracy.
Method: Explainable reasoning analysis system using calibrated, formally verified probes. Models reasoning as function of extractable structure through formal properties: constraint interaction, reasoning depth, solution-space geometry. Generates probes via formal tools with controlled structural variations.
Results: Reveals systematic asymmetry: models robust to constraint refinement but degrade sharply under solution-space restructuring. Calibrated formal probes differentiate models indistinguishable on standard benchmarks and reveal structurally interpretable failure modes.
Conclusion: Provides contamination-free framework supporting training and testing of reasoning models. Enables deeper understanding of LLM reasoning limitations and capabilities beyond surface-level accuracy metrics.

9. A Psyche-Driven Chess Engine with Dynamic Emotional Modulation

arXiv:2603.05352 | Mar 5, 2026
https://arxiv.org/abs/2603.05352
Problem: Chess engines surpass human strength but don't play like humans. Human players exhibit behavioral variability under pressure (tilt, overconfidence) that conventional engines cannot capture.
Method: Personality × psyche decomposition for behavioral variability. Personality is static preset; psyche is dynamic scalar ψ_t ∈ [-100, +100] recomputed from five positional factors after every move. Audio-inspired signal chain (noise gate, compressor/expander, equalizer, limiter) reshapes move probability distributions.
Results: Tested across 12,414 games. Same monotonic gradient in top-move agreement (~20-25 pp spread from stress to overconfidence) across engines differing ~2,800x in training data. Under stress, competitive score fell from 50.8% to 30.1%. Overconfident mode showed 66% agreement with vanilla engine.
Conclusion: Behavioral variation comes from signal chain, not underlying model. Framework produces human-like playing patterns reminiscent of tilt and overconfidence. Interesting exploration of affective computing in game AI.
