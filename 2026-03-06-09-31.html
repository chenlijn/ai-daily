📄 Top 10 arXiv Papers (March 10, 2026)

Agent Training

Agentic Critical Training (ACT)

https://arxiv.org/abs/2603.08706
Problem: Traditional imitation learning for LLM agents only teaches "what to do" without understanding "why," leaving agents unaware of action quality. Even reflection-enhanced methods just imitate pre-constructed reflection text rather than learning autonomous reasoning.
Method: ACT introduces a reinforcement learning paradigm that trains agents to identify the better action among alternatives. By rewarding correct judgment rather than imitation, ACT drives models to autonomously develop reasoning about action quality.
Results: Across three agent benchmarks, ACT achieves average improvements of 5.07 points over imitation learning and 4.62 points over reinforcement learning. Compared to knowledge distillation approaches, ACT yields 2.42 point improvements.
Conclusion: ACT demonstrates that genuine self-reflection can be trained through reinforcement learning rather than injected via distillation, showing strong out-of-distribution generalization and improved general reasoning without task-specific training data.
Agent Evolution

RetroAgent: From Solving to Evolving via Retrospective Dual Intrinsic Feedback

https://arxiv.org/abs/2603.08561
Problem: Standard RL paradigms favor static problem-solving over continuous adaptation. Agents converge to suboptimal strategies due to insufficient exploration, while learned knowledge remains implicit and not explicitly retrievable.
Method: RetroAgent features a hindsight self-reflection mechanism producing dual intrinsic feedback: (1) numerical feedback tracking subtask completion relative to prior attempts, and (2) language feedback distilling reusable lessons into a memory buffer retrieved via Similarity & Utility-Aware UCB strategy.
Results: RetroAgent surpasses GRPO-trained agents by +18.3% on ALFWorld, +15.4% on WebShop, +27.1% on Sokoban, and +8.9% on MineSweeper, achieving state-of-the-art results across four challenging agentic tasks.
Conclusion: The framework demonstrates strong test-time adaptation and out-of-distribution generalization, positioning retrospective self-reflection as a key mechanism for agent evolution beyond static problem-solving.
Multimodal Generation

CoCo: Code as CoT for Text-to-Image Preview and Rare Concept Generation

https://arxiv.org/abs/2603.08652
Problem: Existing Chain-of-Thought methods for text-to-image generation rely on abstract natural language planning, lacking precision for complex spatial layouts, structured visual elements, and dense textual content.
Method: CoCo represents reasoning as executable code that specifies structural layouts, executed in a sandboxed environment to render deterministic draft images. The CoCo-10K dataset contains structured draft-final image pairs teaching both draft construction and visual refinement.
Results: CoCo achieves improvements of +68.83% on StructT2IBench, +54.8% on OneIG-Bench, and +41.23% over direct generation, outperforming other CoT-empowered generation methods.
Conclusion: Executable code is an effective and reliable reasoning paradigm for precise, controllable, and structured text-to-image generation, offering deterministic intermediate representations that natural language cannot match.
Enterprise Benchmark

OfficeQA Pro: An Enterprise Benchmark for End-to-End Grounded Reasoning

https://arxiv.org/abs/2603.08655
Problem: AI agents struggle with grounded, multi-document reasoning over heterogeneous enterprise document corpora. Frontier LLMs achieve less than 5% accuracy when relying on parametric knowledge alone.
Method: OfficeQA Pro introduces a benchmark using U.S. Treasury Bulletins spanning nearly 100 years (89,000 pages, 26M numerical values). Structured document representation via ai_parse_document provides a 16.1% relative performance gain across agents.
Results: With direct document access, frontier agents still struggle on over half of questions, scoring 34.1% average. Claude Opus 4.6, GPT-5.4, and Gemini 3.1 Pro all achieve less than 12% accuracy even with web access.
Conclusion: Significant headroom remains before agents can be considered reliable at enterprise-grade grounded reasoning. The benchmark reveals the gap between frontier LLM capabilities and real-world enterprise document processing needs.
Financial AI

FinToolBench: Evaluating LLM Agents for Real-World Financial Tool Use

https://arxiv.org/abs/2603.08262
Problem: Financial sector AI evaluation focuses on static textual analysis, ignoring the complex reality of tool execution. General tool benchmarks lack domain-specific rigor for high-stakes financial applications.
Method: FinToolBench establishes the first real-world, runnable benchmark with 760 executable financial tools and 295 rigorous tool-required queries. Evaluation assesses agents on timeliness, intent type, and regulatory domain alignment.
Results: The FATR baseline enhances stability and compliance through finance-aware tool retrieval and reasoning. Unlike prior works limited to mock tools, FinToolBench provides auditable agentic financial execution.
Conclusion: FinToolBench sets a new standard for trustworthy AI in finance, providing the first testbed for evaluating real-world financial tool learning agents with actual executable APIs.
Multimodal Reasoning

M3-ACE: Rectifying Visual Perception in Multimodal Math Reasoning via Multi-Agentic Context Engineering

https://arxiv.org/abs/2603.08369
Problem: Multimodal LLM performance in visual math reasoning is limited by inaccurate visual perception. Most failures originate from incorrect visual evidence extraction rather than reasoning deficiencies. Models remain overly confident in initial perceptions.
Method: M3-ACE decouples perception and reasoning by dynamically maintaining a shared context centered on visual evidence lists. Multiple agents collaboratively contribute complementary observations, exposing inconsistencies. Summary and Refine tools organize evidence and filter unreliable samples.
Results: M3-ACE establishes new state-of-the-art results of 89.1 on MathVision benchmark with consistent improvements on MathVista and MathVerse datasets.
Conclusion: Perception-centric multi-agent collaboration is essential for advancing multimodal reasoning systems. The work highlights that visual perception quality, not just reasoning capability, limits multimodal AI performance.
Trust & Safety

Trust via Reputation of Conviction

https://arxiv.org/abs/2603.08575
Problem: AI agents are capable but error-prone sources requiring robust trust frameworks. Current approaches focus on correctness or faithfulness, which are regime-dependent and fail to reward genuine contribution.
Method: The paper defines truth as reproducibly perceived knowledge and formalizes sources as having generative and discriminative roles. Reputation is grounded in "conviction"—the likelihood a source's stance is vindicated by independent consensus. Continuous verification is identified as both theoretical necessity and practical mechanism.
Results: The framework characterizes reputation behavior across source-claim regimes and identifies that verifiable conviction and continuously accrued reputation constitute the only robust foundation for trust in AI agents.
Conclusion: Conviction rather than correctness is the principled basis for trust: it is regime-independent, rewards genuine contribution, and demands transparent perceptions enabling external verification.
AI Assistant Architecture

IronEngine: Towards General AI Assistant

https://arxiv.org/abs/2603.08425
Problem: General AI assistants require integration of desktop UI, APIs, model backends, persistent memory, task scheduling, skills, and tool execution without compromising planning quality or execution capability.
Method: IronEngine introduces a three-phase pipeline (Discussion, Model Switch with VRAM-aware transition, Execution) separating planning from execution. Features include hierarchical memory with multi-level consolidation, vectorized skill repository via ChromaDB, adaptive model management with 92 model profiles, and intelligent tool routing with 130+ alias normalization.
Results: File operation benchmarks achieve 100% task completion with mean total time of 1541 seconds across four heterogeneous tasks. Detailed comparisons with ChatGPT, Claude Desktop, Cursor, and Windsurf provided.
Conclusion: IronEngine positions itself as a system-oriented foundation for general-purpose personal assistants, automation frameworks, and human-centered agent platforms through architectural decomposition and subsystem modularity.
Tool Learning

In-Context Reinforcement Learning for Tool Use in Large Language Models

https://arxiv.org/abs/2603.08068
Problem: Enabling LLMs to use external tools effectively typically requires supervised fine-tuning followed by RL, demanding substantial labeled data that is expensive to annotate or synthesize.
Method: ICRL is an RL-only framework eliminating the need for SFT by leveraging few-shot prompting during rollout. In-context examples teach models how to invoke tools, with gradual reduction to zero-shot setting where models learn independent tool calling.
Results: ICRL achieves state-of-the-art performance across reasoning and tool-use benchmarks, demonstrating effectiveness as a scalable, data-efficient alternative to traditional SFT-based pipelines.
Conclusion: Tool use capabilities can be learned purely through in-context reinforcement learning without expensive supervised pre-training, opening pathways for more efficient agent development.
AI Safety

The Struggle Between Continuation and Refusal: Mechanistic Analysis of Continuation-Triggered Jailbreak

https://arxiv.org/abs/2603.08234
Problem: Despite safety alignment efforts, LLMs remain vulnerable to jailbreaking attacks. The root causes of such vulnerabilities remain poorly understood at the mechanistic level.
Method: Comprehensive mechanistic interpretability analysis at the attention head level reveals that continuation-triggered jailbreaks arise from competition between intrinsic continuation drive and safety defenses acquired through alignment training. Causal interventions and activation scaling identify safety-critical attention heads.
Results: Analysis reveals notable functional and behavioral differences in safety heads across model architectures. Relocating continuation-triggered instruction suffixes substantially increases jailbreak success rates.
Conclusion: The findings provide a novel mechanistic perspective for understanding jailbreak behaviors, offering theoretical insights and practical implications for improving model safety through attention head-level interventions.
