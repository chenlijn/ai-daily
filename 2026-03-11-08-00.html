📄 Top 10 arXiv Papers

Time window: March 10, 2026 | 10 selected papers with Problem/Method/Results/Conclusion

1. Agentic Critical Training (ACT)
arXiv:2603.08706
https://arxiv.org/abs/2603.08706
Problem: Traditional imitation learning for LLM agents only teaches "what to do" without understanding "why." Agents lack awareness of action quality.
Method: ACT introduces RL paradigm training agents to identify the better action among alternatives. Rewards correct judgment rather than imitation.
Results: Average improvements of 5.07 points over imitation learning and 4.62 over standard RL. Strong out-of-distribution generalization.
Conclusion: Genuine self-reflection can be trained through RL, enabling agents to autonomously reason about action quality.
2. RetroAgent: From Solving to Evolving via Retrospective Dual Intrinsic Feedback
arXiv:2603.08561
https://arxiv.org/abs/2603.08561
Problem: Standard RL paradigms favor static problem-solving over continuous adaptation. Agents converge to suboptimal strategies.
Method: Hindsight self-reflection with dual intrinsic feedback: numerical feedback tracking subtask completion and language feedback distilling lessons into memory.
Results: Surpasses GRPO by +18.3% on ALFWorld, +15.4% on WebShop, +27.1% on Sokoban, +8.9% on MineSweeper.
Conclusion: Retrospective self-reflection enables agent evolution beyond static problem-solving with strong test-time adaptation.
3. CoCo: Code as CoT for Text-to-Image Preview and Rare Concept Generation
arXiv:2603.08652
https://arxiv.org/abs/2603.08652
Problem: Existing CoT methods for T2I rely on abstract natural language planning, lacking precision for complex spatial layouts.
Method: CoCo represents reasoning as executable code specifying structural layouts. Generates deterministic draft images from code execution.
Results: Improvements of +68.83% on StructT2IBench, +54.8% on OneIG-Bench, +41.23% over direct generation.
Conclusion: Executable code is superior to natural language for precise, controllable text-to-image generation.
4. OfficeQA Pro: Enterprise Benchmark for End-to-End Grounded Reasoning
arXiv:2603.08655
https://arxiv.org/abs/2603.08655
Problem: AI agents struggle with grounded multi-document reasoning over heterogeneous enterprise corpora. Frontier LLMs achieve less than 5% accuracy.
Method: Benchmark using U.S. Treasury Bulletins spanning 100 years (89,000 pages, 26M values). Evaluated Claude Opus 4.6, GPT-5.4, Gemini 3.1 Pro.
Results: With direct document access, frontier agents score 34.1% average. Structured document representation yields 16.1% relative gain.
Conclusion: Significant headroom remains before agents can be considered reliable at enterprise-grade grounded reasoning.
5. FinToolBench: Evaluating LLM Agents for Real-World Financial Tool Use
arXiv:2603.08262
https://arxiv.org/abs/2603.08262
Problem: Financial AI evaluation focuses on static textual analysis, ignoring complex tool execution reality.
Method: First real-world benchmark with 760 executable financial tools and 295 tool-required queries.
Results: FATR baseline enhances stability and compliance through finance-aware retrieval and reasoning.
Conclusion: Sets new standard for trustworthy AI in finance with auditable agentic execution.
6. M3-ACE: Rectifying Visual Perception in Multimodal Math Reasoning
arXiv:2603.08369
https://arxiv.org/abs/2603.08369
Problem: Multimodal LLM performance limited by inaccurate visual perception. Most failures from incorrect visual evidence extraction.
Method: Decouples perception and reasoning via shared context on visual evidence lists. Multi-agent collaboration exposes inconsistencies.
Results: New SOTA of 89.1 on MathVision with consistent improvements on MathVista and MathVerse.
Conclusion: Perception-centric multi-agent collaboration is essential for multimodal reasoning advancement.
7. Trust via Reputation of Conviction
arXiv:2603.08575
https://arxiv.org/abs/2603.08575
Problem: AI agents are capable but error-prone requiring robust trust frameworks.
Method: Defines truth as reproducibly perceived knowledge. Reputation grounded in "conviction"—likelihood source's stance is vindicated by consensus.
Results: Verifiable conviction and continuously accrued reputation constitute the only robust foundation for trust.
Conclusion: Conviction rather than correctness is principled basis for trust—regime-independent and rewarding genuine contribution.
8. IronEngine: Towards General AI Assistant
arXiv:2603.08425
https://arxiv.org/abs/2603.08425
Problem: General AI assistants require integration of UI, APIs, memory, scheduling, skills without compromising planning or execution.
Method: Three-phase pipeline (Discussion, Model Switch, Execution) separating planning from execution. Hierarchical memory, ChromaDB skills, 92 model profiles.
Results: 100% task completion in 1541s mean time across four heterogeneous file operation tasks.
Conclusion: System-oriented foundation for general-purpose assistants through architectural decomposition.
9. In-Context Reinforcement Learning for Tool Use in LLMs
arXiv:2603.08068
https://arxiv.org/abs/2603.08068
Problem: Enabling LLM tool use typically requires expensive SFT followed by RL with substantial labeled data.
Method: ICRL is RL-only framework eliminating SFT need via few-shot prompting during rollout. Gradual reduction to zero-shot tool calling.
Results: State-of-the-art performance across reasoning and tool-use benchmarks as scalable, data-efficient alternative.
Conclusion: Tool use can be learned purely through in-context RL without expensive supervised pre-training.
10. The Struggle Between Continuation and Refusal: Mechanistic Analysis of Jailbreak
arXiv:2603.08234
https://arxiv.org/abs/2603.08234
Problem: LLMs remain vulnerable to jailbreaking; root causes poorly understood at mechanistic level.
Method: Comprehensive mechanistic interpretability at attention head level. Causal interventions and activation scaling.
Results: Jailbreaks arise from competition between intrinsic continuation drive and safety defenses.
Conclusion: Novel mechanistic perspective for understanding jailbreak behaviors with implications for safety improvements.
