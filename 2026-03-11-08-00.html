📄 Top 10 arXiv Papers (March 10, 2026)

时间窗口：北京时间3月9日早8点 - 3月10日早8点 | 10篇精选论文深度分析

1. Agentic Critical Training (ACT)
https://arxiv.org/abs/2603.08706
问题：传统模仿学习只教智能体"做什么"而不解释"为什么"，智能体缺乏对行动质量的认知，因为他们从未对比成功行动和次优替代方案。
方法：ACT引入强化学习范式，训练智能体在备选方案中识别更优行动。奖励正确的判断力而非模仿行为，模型发展出真正的自我反思能力而非模仿预构建的反思文本。
结果：ACT相比模仿学习平均提升5.07分，相比标准RL提升4.62分。与知识蒸馏方法相比提升2.42分，具有强大的分布外泛化能力。
结论：ACT为开发更具反思能力和能力的LLM智能体提供了有前景的路径，使智能体能够真正自主地推理行动质量。
2. RetroAgent: From Solving to Evolving via Retrospective Dual Intrinsic Feedback
https://arxiv.org/abs/2603.08561
问题：标准RL范式偏好静态问题解决而非持续适应；智能体因探索不足收敛到次优策略，隐性知识无法检索。
方法：RetroAgent具有后见之明自我反思功能，产生双内在反馈：（1）跟踪子任务完成的数值反馈，（2）将经验教训提炼为语言反馈并存入记忆。使用SimUtil-UCB策略平衡相关性、实用性和探索。
结果：超越GRPO：ALFWorld提升18.3%，WebShop提升15.4%，Sokoban提升27.1%，MineSweeper提升8.9%。具有强大的测试时适应和OOD泛化能力。
结论：通过后见反馈和显式记忆，智能体不仅能解决问题，还能通过反思反馈不断进化。
3. CoCo: Code as CoT for Text-to-Image Preview and Rare Concept Generation
https://arxiv.org/abs/2603.08652
问题：现有T2I的CoT方法依赖抽象自然语言规划，缺乏复杂空间布局、结构化视觉元素和密集文本内容的精确性。
方法：CoCo（Code-as-CoT）使用可执行代码表示推理过程。生成指定场景布局的代码，在沙箱环境中执行以渲染确定性草稿图像，然后通过细粒度编辑进行精炼。构建CoCo-10K数据集。
结果：StructT2IBench提升68.83%，OneIG-Bench提升54.8%，LongText-Bench相比直接生成提升41.23%。优于其他CoT增强生成方法。
结论：可执行代码是精确、可控、结构化文本到图像生成的有效可靠推理范式。代码：https://github.com/micky-li-hd/CoCo
4. OfficeQA Pro: An Enterprise Benchmark for End-to-End Grounded Reasoning
https://arxiv.org/abs/2603.08655
问题：AI智能体难以在异构企业文档语料库上进行grounded多文档推理。前沿LLM使用参数知识准确率不足5%。
方法：使用跨越近100年的美国财政部公告（8.9万页，2600万+数值），133个问题需要精确文档解析、检索和分析推理。评估Claude Opus 4.6、GPT-5.4、Gemini 3.1 Pro。
结果：直接使用文档语料库，前沿智能体平均得分34.1%，一半以上问题 struggled。Claude/GPT/Gemini网络访问准确率不足12%。结构化文档表示提升16.1%相对性能。
结论：智能体在企业级grounded推理方面达到可靠水平仍有巨大提升空间。结构化文档表示至关重要。
5. FinToolBench: Evaluating LLM Agents for Real-World Financial Tool Use
https://arxiv.org/abs/2603.08262
问题：金融AI评估聚焦静态文本分析，忽略复杂工具执行现实。通用基准缺乏领域特定严谨性。
方法：首个真实世界基准，包含760个可执行金融工具和295个需要工具的查询。评估时效性、意图类型、监管一致性。
结果：FATR基线通过金融感知检索和推理增强稳定性和合规性。
结论：为金融领域可信赖AI设定新标准，具备可审计的agent执行能力。
6. M3-ACE: Rectifying Visual Perception in Multimodal Math Reasoning
https://arxiv.org/abs/2603.08369
问题：多模态LLM性能受不准确的视觉感知限制。大多数失败源于不正确的视觉证据提取，而非推理缺陷。
方法：通过视觉证据列表上的共享上下文解耦感知和推理。多智能体协作暴露不一致性。摘要和精炼工具组织/过滤证据。
结果：MathVision新SOTA达89.1，MathVista和MathVerse持续改进。
结论：以感知为中心的多智能体协作对多模态推理进步至关重要。
7. Trust via Reputation of Conviction
https://arxiv.org/abs/2603.08575
问题：AI智能体有能力但容易出错，需要稳健的信任框架。当前方法聚焦正确性，可能不是regime-independent。
方法：将知识、真理和信任数学形式化。开发基于"信念"的声誉框架——即源立场被独立共识证实的概率。
结果：信念是regime-independent的，奖励真正贡献，要求透明自足的感知实现外部验证。
结论：对于AI智能体（有能力但容易出错的源），可验证的信念和持续积累的声誉构成信任的唯一稳健基础。
8. IronEngine: Towards General AI Assistant
https://arxiv.org/abs/2603.08425
问题：通用AI助手需要集成UI、API、内存、调度、技能，不损害规划或执行。
方法：三阶段管道（讨论、模型切换、执行）分离规划与执行。分层内存，ChromaDB技能库，92个模型配置文件，130+工具别名规范化。
结果：文件操作基准测试：四个异构任务中100%任务完成，平均1541秒。
结论：通过架构分解定位为通用个人助手、自动化框架和未来以人为中心的Agent平台的系统导向基础。
9. In-Context Reinforcement Learning for Tool Use in LLMs
https://arxiv.org/abs/2603.08068
问题：启用LLM工具使用通常需要昂贵的SFT，然后使用大量标记数据的RL。
方法：ICRL是纯RL框架，通过rollout期间的few-shot提示消除SFT需求。逐步减少到zero-shot工具调用。
结果：推理和工具使用基准测试中达到SOTA性能，作为可扩展、数据高效的替代方案。
结论：工具使用可以纯粹通过in-context RL学习，无需昂贵的监督预训练。
10. The Struggle Between Continuation and Refusal: Mechanistic Analysis of Jailbreak
https://arxiv.org/abs/2603.08234
问题：LLM仍然容易受到越狱攻击；在机制层面根本原因 poorly understood。
方法：在注意力头层面进行全面的机械可解释性。因果干预和激活缩放。
结果：越狱源于内在延续驱动和安全防御之间的竞争。不同架构的安全头存在显著功能差异。
结论：理解越狱行为的新机制视角，对安全改进具有启示意义。
