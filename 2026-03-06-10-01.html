📄 Top 10 arXiv Papers (March 5-6, 2026)

1. KARL: Knowledge Agents via Reinforcement Learning

https://arxiv.org/abs/2603.05218
Problem: Enterprise search requires agents capable of complex, multi-step reasoning across diverse information sources with limited verifiable rewards.
Method: Introduced KARLBench with 6 search regimes, agentic synthesis pipeline for training data generation, and iterative large-batch off-policy RL paradigm that is sample-efficient and robust to train-inference discrepancies.
Results: KARL achieves Pareto-optimal performance vs Claude 4.6 and GPT 5.2 across cost-quality and latency-quality trade-offs, with out-of-distribution generalization capabilities.
Conclusion: Tailored synthetic data combined with multi-task RL enables cost-efficient, high-performing knowledge agents for grounded reasoning in enterprise settings.

2. WebChain: Large-Scale Human-Annotated Dataset for Web Agents

https://arxiv.org/abs/2603.05295
Problem: Web agents lack large-scale, high-quality datasets of real-world human interactions on actual websites for training and evaluation.
Method: Collected 31,725 trajectories with 318K steps featuring Triple Alignment of visual, structural, and action data. Proposed Dual Mid-Training recipe decoupling spatial grounding from planning.
Results: Achieved state-of-the-art performance on WebChainBench and other public GUI benchmarks through the novel training approach.
Conclusion: Real-world human-annotated data with multi-modal supervision is essential for building next-generation scalable web agents.

3. STRUCTUREDAGENT: Planning with AND/OR Trees for Long-Horizon Web Tasks

https://arxiv.org/abs/2603.05294
Problem: Web agents struggle with complex long-horizon tasks due to limited in-context memory, weak planning abilities, and greedy behaviors causing premature termination.
Method: Hierarchical planning framework with online AND/OR tree planner for efficient search and structured memory module tracking candidate solutions for constraint satisfaction.
Results: Improved performance on WebVoyager, WebArena, and custom shopping benchmarks compared to standard LLM-based agents.
Conclusion: Explicit hierarchical planning with interpretable structures enables better debugging and human intervention for web agents.

4. OPENDEV: Building AI Coding Agents for the Terminal

https://arxiv.org/abs/2603.05344
Problem: CLI-based coding agents need safety controls and efficient context management to prevent context bloat and reasoning degradation in long-horizon tasks.
Method: Compound AI system with workload-specialized model routing, dual-agent architecture (planning vs execution), lazy tool discovery, adaptive context compaction, and automated memory system.
Results: Provides secure, extensible foundation for terminal-first AI assistance with event-driven system reminders to counteract instruction fade-out.
Conclusion: Terminal-native agents operating directly in developer workflows offer unprecedented autonomy for software engineering tasks.

5. Judge Reliability Harness: Stress Testing LLM Judges

https://arxiv.org/abs/2603.05399
Problem: LLM-based scoring is widely deployed but lacks tools to assess reliability, with judges showing consistency issues under simple perturbations.
Method: Open-source library generating reliability tests for binary judgment accuracy and ordinal grading across free-response and agentic task formats.
Results: No evaluated judge was uniformly reliable across benchmarks. Exposed consistency issues from formatting changes, paraphrasing, verbosity changes, and label flipping.
Conclusion: Critical need for standardized reliability testing before deploying LLM judges in production AI systems.

6. Towards Provably Unbiased LLM Judges via Bias-Bounded Evaluation

https://arxiv.org/abs/2603.05485
Problem: Autonomous AI systems need automated verifiable rewards, but LLM judges lack strong guarantees when bias vectors are unknown or adversarial.
Method: Proposed average bias-boundedness (A-BB) algorithmic framework formally guaranteeing harm reduction from any measurable bias.
Results: Achieved (tau=0.5, delta=0.01) bias-bounded guarantees while retaining 61-99% correlation with original rankings across formatting and schematic bias settings.
Conclusion: Formal bias-bounded frameworks can enable trustworthy LLM-as-a-Judge systems for autonomous AI feedback loops.

7. The Spike, the Sparse and the Sink: Anatomy of Massive Activations and Attention Sinks

https://arxiv.org/abs/2603.05498
Problem: Massive activations and attention sinks frequently co-occur in Transformers but their functional roles and causal relationship remain unclear.
Method: Systematic experiments showing co-occurrence is largely an architectural artifact of pre-norm configuration, with massive activations operating globally and attention sinks operating locally.
Results: Massive activations induce near-constant hidden representations functioning as implicit parameters, while attention sinks modulate outputs across heads biasing toward short-range dependencies.
Conclusion: Pre-norm configuration enables co-occurrence; ablating it causes the two phenomena to decouple, providing insights for Transformer architecture design.

8. X-RAY: Mapping LLM Reasoning via Formalized and Calibrated Probes

https://arxiv.org/abs/2603.05290
Problem: LLM reasoning capability is poorly understood with evaluations conflating pattern matching with true reasoning; need for explainable analysis.
Method: Explainable reasoning analysis system using calibrated, formally verified probes modeling reasoning as function of extractable structure through properties like constraint interaction and reasoning depth.
Results: Revealed systematic asymmetry: models robust to constraint refinement but degrade sharply under solution-space restructuring. Formal probes differentiate models indistinguishable on standard benchmarks.
Conclusion: Contamination-free formal probes enable both evaluation and training of reasoning models with structurally interpretable failure modes.

10. MedCoRAG: Interpretable Hepatology Diagnosis via Hybrid Evidence Retrieval

https://arxiv.org/abs/2603.05129
Problem: Clinical AI lacks transparency, structured reasoning, and deployability; single-source retrieval fails to support iterative, role-specialized deliberation.
Method: Medical Collaborative RAG framework retrieving from UMLS knowledge graph AND clinical guidelines with Multi-Agent Collaborative Reasoning: Router Agent dispatches Specialist Agents based on case complexity.
